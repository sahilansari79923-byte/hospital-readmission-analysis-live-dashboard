# Hospital Patient Readmission Analysis

This project looks at what makes diabetic patients more likely to be readmitted to hospital within 30 days of discharge. The dataset covers 130 US hospitals across 1999–2008, and has around 100,000 patient records with 50 features.

I built this as a portfolio project while transitioning into data analytics. The pipeline goes from raw data all the way to a trained classifier, with SQL queries and charts in between.

## Dataset

**Source:** [UCI Machine Learning Repository — Diabetes 130-US Hospitals](https://archive.ics.uci.edu/dataset/296/diabetes+130-us+hospitals+for+years+1999-2008)

The raw CSV isn't included because it's ~23MB and git doesn't love that. Download it from the link above and put it in `data/raw/diabetic_data.csv`.

Quick stats on the raw file:
- 101,766 rows, 50 columns
- Missing values are encoded as `?` (not NaN)
- Target column: `readmitted` — values are `NO`, `>30`, `<30`

## What I found

**Overall readmission rate: 8.84%** — out of 69,560 cleaned patient records, 6,150 were readmitted within 30 days.

A few things stood out:

**Age matters a lot.** Patients in the 80–90 age bracket had a 10.36% readmission rate, more than double the rate of patients under 40. Not surprising, but the jump really kicks in after 60.

**Insulin dosage changes are a red flag.** Patients whose insulin was decreased (`Down`) had the highest readmission rate at 10.42%, compared to 8.17% for patients on no insulin at all. It makes sense — a dose reduction that goes wrong means unstable glucose, which means a quick return trip.

**More diagnoses = higher risk.** Patients with 9+ diagnoses had a ~9.8% readmission rate versus 3.2% for patients with just one. The more conditions someone has, the harder discharge management gets.

**High medication counts also correlate with readmission.** Patients on 20+ medications had a 10.24% readmission rate versus 7.04% for those on fewer than 10. This probably reflects clinical complexity more than the medications themselves.

## Model results

I trained two models: Logistic Regression and Random Forest. Both used `class_weight='balanced'` because the data is ~9:1 imbalanced.

| Model | Accuracy | ROC-AUC |
|---|---|---|
| Logistic Regression | 61.2% | 0.602 |
| Random Forest | 90.7% | 0.554 |

Random Forest's 90% accuracy is misleading — it achieves that mostly by predicting "not readmitted" for almost everyone. Logistic Regression's ROC-AUC is actually higher (0.602 vs 0.554), meaning it does a better job telling the two groups apart despite lower raw accuracy.

**Top 5 features (Random Forest):**
1. Number of lab procedures — 0.319
2. Number of medications — 0.236
3. Time in hospital — 0.120
4. Age — 0.094
5. Number of diagnoses — 0.089

## Limitations

A1Cresult and max_glu_serum were dropped because over 40% of values were missing. These are probably the most clinically relevant features for a diabetic readmission study, so losing them hurts. Better data collection at hospitals would make a real difference here.

The class imbalance (91:9) is the other main issue. SMOTE or other resampling techniques would likely improve recall on the minority class. That's the next thing I'd try.
